67  /*
68  * The user or runtime system can invoke this method at any time (when a valid
69  * pointer to it exists) to migrate this VCC element to another memory domain
70  *
71  * 1. Invoke migrate(node) where node != theContext()->getNode()
72  * 2. Runtime system invokes Migratable::preMigrateOut()
73  * 3. Migratable element is serialized
74  * 4. Migratable element is sent to the destination node
75  * 5. Location manager de-registers this element
76  * 6. VCC manager removes local reference to this element
77  * 7. Runtime system invokes Migratable::epiMigrateOut()
78  * 8. Runtime system invokes Migratable::destroy()
79  * 9. Runtime system invokes the destructor
80  * ....... send MigrateMsg to MigrateHandlers::migrateInHandler .......
81  * 10. migrateInHandler() is invoked
82  * 11. Migratable element is de-serialized
83  * 12. Migratable element constructed with migration constructor
84  * 13. Runtime system invokes Migratable::preMigrateIn()
85  * 14. VCC manager add local reference to this element (InnerHolder)
86  * 15. Location manager registers this element on destination node
87  * 16. Runtime system invokes Migratable::epiMigrateIn()
88  *
89  */
