127  /*
128  * Get the location of a entity: the `action' is triggered when the location
129  * of the entity is resolved. This is an asynchronous call that may send
130  * messages to discover the location of the entity `id'. To resolve the
131  * location the method uses the following algorithm:
132  *
133  * 1) Check locally for the entity's existence
134  *. 2) If not local, search for a cache entry with location info
135  * 3) If no cache information available, send resolution message to home node.
136  * a) The home node applies the same algorithm, starting with (1)
137  *. . b) On step 3, if no information is known, the manager buffers the
138  *. request, waiting to the entity to be registered in the future.
139  *
140  * Note: migrations may make this information inaccurate; the node delivered
141  * to `action' reflects the current known state, which may be remote.
